Connecting more than one Domain

If you want to run more than one website on your web server, you can, and you’ll also be able to connect more than one domain to these websites.

Before you continue reading along, please read Connecting your Domain(Cloudflare), as you’ll need to learn how to setup a Cloudflare worker to each domains.

For an example, we’ll setup two websites on our server. Steps can be repeated if you’ve more than two websites. We’ll call them both Website 1 and Website 2 through out this article. At the root of our file manager, we should go ahead and create a folder for website 1 and website 2:

/home/container/website-1
                               /website-2

Now we need to tell Nginx that we want a specific domain for website 1 and a different domain for website 2.
We do can this by configuring the default file located in `/home/container/nginx/sites-available/default.conf`

You should see a lot of stuff in server {}. Just change the file path to website 1 and set server_name as the domain you’ll use for website 1.
Like server_name domain_1.com;

Here’s some code as an example:

server {
                    listen 25000;
                    server_name domain1.com;
                
                    root /home/container/website-1;
                    index index.html index.htm index.php;
                    charset utf-8;
                
                    location / {
                        try_files $uri $uri/ /index.html;
                    }
                
                    # allow larger file uploads and longer script runtimes
                    client_max_body_size 100m;
                    client_body_timeout 120s;
                
                    sendfile off;
                
                    location ~ \.php$ {
                        fastcgi_split_path_info ^(.+\.php)(/.+)$;
                        fastcgi_pass unix:/tmp/php-fpm.sock;
                        fastcgi_index index.php;
                        include fastcgi_params;
                        fastcgi_param PHP_VALUE "upload_max_filesize = 100M \n post_max_size=100M";
                        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
                        fastcgi_param HTTP_PROXY "";
                        fastcgi_intercept_errors off;
                        fastcgi_buffer_size 16k;
                        fastcgi_buffers 4 16k;
                        fastcgi_connect_timeout 300;
                        fastcgi_send_timeout 300;
                        fastcgi_read_timeout 300;
                    }
                
                    location ~ /\.ht {
                        deny all;
                    }
                }
                
                server {
                    listen 25000;
                    server_name domain2.com;
                
                    root /home/container/website-2;
                    index index.html index.htm index.php;
                    charset utf-8;
                
                    location / {
                        try_files $uri $uri/ /index.html;
                    }
                
                    # allow larger file uploads and longer script runtimes
                    client_max_body_size 100m;
                    client_body_timeout 120s;
                
                    sendfile off;
                
                    location ~ \.php$ {
                        fastcgi_split_path_info ^(.+\.php)(/.+)$;
                        fastcgi_pass unix:/tmp/php-fpm.sock;
                        fastcgi_index index.php;
                        include fastcgi_params;
                        fastcgi_param PHP_VALUE "upload_max_filesize = 100M \n post_max_size=100M";
                        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
                        fastcgi_param HTTP_PROXY "";
                        fastcgi_intercept_errors off;
                        fastcgi_buffer_size 16k;
                        fastcgi_buffers 4 16k;
                        fastcgi_connect_timeout 300;
                        fastcgi_send_timeout 300;
                        fastcgi_read_timeout 300;
                    }
                
                    location ~ /\.ht {
                        deny all;
                    }
                }

If one of your websites require more information in the Nginx config file, add them as needed.
No, you don’t need to use more ports. You’re allowed to use the same port on as many websites as you want. So website 1 and website 2 can use the same port.
